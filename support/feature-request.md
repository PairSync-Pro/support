# Request a Feature

Have an idea to make PairSync better? We'd love to hear it! User feedback directly shapes our product roadmap.

---

## How to Submit a Feature Request

**Email:** mark@pair-sync.com  
**Subject:** "Feature Request: [Brief Description]"

Example: "Feature Request: Add push notifications for signal alerts"

---

## Feature Request Template

**Please use this template:**

```
FEATURE REQUEST

1. FEATURE SUMMARY
Brief description (1-2 sentences):


2. DETAILED DESCRIPTION
What exactly would this feature do?


3. USE CASE
What problem does this solve? How would you use it?


4. EXAMPLE SCENARIO
Walk us through a specific example:
"I would open PairSync, navigate to..., then I would..."


5. PRIORITY FOR YOU
□ Critical (can't use app effectively without it)
□ High (would significantly improve my experience)
□ Medium (nice to have)
□ Low (minor improvement)


6. ALTERNATIVE SOLUTIONS
Are there workarounds you're currently using?


7. SIMILAR FEATURES IN OTHER APPS
Have you seen this feature elsewhere? (helps us understand)


8. YOUR USE CASE FREQUENCY
How often would you use this feature?
□ Daily □ Weekly □ Monthly □ Rarely


9. WILLING TO BETA TEST?
□ Yes, I'd love to test this feature early
□ No, just sharing the idea
```

## How We Evaluate Requests

**We consider several factors:**

1. **User Demand**  
   How many users are asking for this? (Vote with your requests!)

2. **Alignment with Mission**  
   Does this support education and learning? (Our core mission)

3. **Technical Feasibility**  
   Can we build this with available resources and technology?

4. **Cost vs Value**  
   What's the development cost vs benefit to users?

5. **Legal/Compliance**  
   Does this create regulatory concerns? (e.g., brokerage integration is complex)

6. **Maintenance Burden**  
   Will this require ongoing maintenance or third-party dependencies?

---

## What Happens After You Submit

1. **Acknowledgment (48-72 hours)**  
   We'll confirm receipt and may ask clarifying questions.

2. **Review**  
   Product team reviews and assesses feasibility.

3. **Prioritization**  
   Feature is added to our roadmap with a priority level.

4. **Development Timeline**  
   - **High priority:** 1-3 months
   - **Medium priority:** 3-6 months
   - **Low priority:** 6-12+ months or "future consideration"

5. **Updates**  
   We may email you when the feature is in development or released.

6. **Testing (if applicable)**  
   You may be invited to beta test if you indicated interest.

## Feature Request Examples

### Great Example ✅

**Subject:** Feature Request: Export watchlist to CSV

**Body:**
```
FEATURE SUMMARY:
Ability to export my watchlist pairs to a CSV file.

DETAILED DESCRIPTION:
Add an "Export" button in the Watchlist screen that generates a CSV file with columns:
- Symbol 1
- Symbol 2
- Current Z-Score
- Sharpe Ratio
- Market Geography
- Asset Class

PROBLEM THIS SOLVES:
I want to analyze my watchlist pairs in Excel for custom calculations and record-keeping. Currently, I have to manually copy data.

USE CASE:
As an active researcher, I track pairs in a spreadsheet. Exporting would save me 30 minutes weekly.

EXAMPLE SCENARIO:
1. I open Watchlist tab
2. Tap "..." menu (top right)
3. Select "Export to CSV"
4. File saves to Files app or shares via email
5. I open in Excel/Google Sheets

PRIORITY FOR ME:
High - would significantly improve my experience

ALTERNATIVE SOLUTIONS:
Currently, I manually type each pair into Excel (tedious).

SIMILAR FEATURES:
Similar to "Export" in banking apps for transaction history.

FREQUENCY:
Weekly

WILLING TO BETA TEST:
Yes, I'd love to test this!
```

**Why this is great:** Specific, clear use case, detailed scenario, shows value.

## Features We Can't Implement

**For legal, technical, or strategic reasons, we CANNOT:**

❌ **Execute trades or integrate with brokerages**  
Reason: Would require registering as a broker-dealer (massive regulatory burden)

❌ **Provide personalized investment advice**  
Reason: Would require SEC registration as an investment advisor

❌ **Guarantee profitability or specific returns**  
Reason: Illegal and impossible (markets are unpredictable)

❌ **Build features that violate Apple App Store guidelines**  
Reason: Would get us removed from the App Store

❌ **Access data we don't have rights to**  
Reason: Data licensing restrictions

**We appreciate all ideas, but some simply aren't feasible.**
